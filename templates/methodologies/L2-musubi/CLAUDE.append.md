### L2-musubi — methodology notes

This project uses **MUSUBI** (L2 Spec-Anchored with EARS + C4 + ADR + traceability matrix).

**The spec lives the whole project lifecycle.** Every change requires an addendum that propagates to design, code, and tests. Not "spec then code" — "spec + design + code + tests continuously aligned."

**Workflow for any new feature:**

1. Invoke `@requirements-analyst` — write requirements in EARS format with REQ-DOMAIN-NNN identifiers
2. Invoke `@system-architect` — produce C4 diagrams (Context / Container / Component / Code) and ADR if architectural
3. Invoke `@test-engineer` — write tests *before* implementation (Article III is blocking)
4. Implement
5. `@traceability-auditor` runs and blocks merge if REQ→Design→Code→Test coverage isn't 100%
6. `@constitution-enforcer` checks every file before merge — violations of Articles III, IV, V block

**Constitution (live document, every change propagates):** see `docs/constitution.md`. Critical (merge-blocking) articles:

- **Article III — Test-First Imperative** (mandatory TDD enforcement)
- **Article IV — EARS Format** (no SHOULD/MAY in requirements)
- **Article V — 100% Traceability** (every REQ has a Design, Code, Test link)

**EARS quick reference:**

```
REQ-AUTH-001 [Ubiquitous]:
  The system SHALL hash all passwords using bcrypt with cost factor ≥ 12.

REQ-AUTH-002 [Event-Driven]:
  WHEN user submits login form with valid credentials,
  the system SHALL return JWT token valid for 15 minutes.

REQ-AUTH-003 [Unwanted Behaviour]:
  IF login fails 5 times within 15 minutes,
  THEN the system SHALL lock the account for 30 minutes.
```

**Traceability matrix lives at:** `docs/traceability.md` (generated by `@traceability-auditor`). Format:

```
REQ-AUTH-001 → design/auth-component.md § 2 → src/auth/service.ts:45 → tests/auth.test.ts:12
```

**Brownfield within this preset:** if you're adding to existing code, run `@change-impact-analyzer` first — it generates delta specs (ADDED / MODIFIED / REMOVED) relative to the current state.

**When NOT to use the full ceremony:**
- One-line fixes that don't change behavior: skip the full flow, log in addendum log.
- Pure refactors with no behavior change: ADR-only.
- Spike / proof-of-concept on a branch that will never merge: lean preset for that branch.
